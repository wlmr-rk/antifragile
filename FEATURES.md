## Database Schema

### Todos Table

| Field | Type | Description |
|-------|------|-------------|
| `_id` | ID | Auto-generated unique identifier |
| `text` | string | The todo content/description |
| `isCompleted` | boolean | Completion status |
| `createdAt` | number | Unix timestamp of creation |
| `_creationTime` | number | Auto-generated by Convex |

### Indexes
- **`by_creation_time`** - Index on `createdAt` for efficient sorting

## API Functions

### Queries

#### `getTodos()`
- **Purpose:** Fetch all todos
- **Arguments:** None
- **Returns:** Array of todos ordered by creation time (newest first)
- **Use case:** Display todo list

### Mutations

#### `addTodo(text: string)`
- **Purpose:** Create a new todo
- **Arguments:**
  - `text` - The todo description
- **Returns:** ID of created todo
- **Use case:** Adding new tasks

#### `toggleTodo(id: Id<"todos">)`
- **Purpose:** Toggle completion status
- **Arguments:**
  - `id` - Todo identifier
- **Returns:** Nothing
- **Use case:** Marking tasks as complete/incomplete

#### `deleteTodo(id: Id<"todos">)`
- **Purpose:** Remove a todo
- **Arguments:**
  - `id` - Todo identifier
- **Returns:** Nothing
- **Use case:** Deleting unwanted tasks

#### `clearCompleted()`
- **Purpose:** Delete all completed todos
- **Arguments:** None
- **Returns:** Nothing
- **Use case:** Bulk cleanup of finished tasks

## User Flows

### Adding a Todo
1. User types in the input field
2. User clicks "Add" or presses Enter
3. Form validation checks for empty input
4. Mutation sends data to Convex
5. Convex creates new todo in database
6. Real-time query updates automatically
7. New todo appears in list instantly
8. Input field clears for next entry

### Completing a Todo
1. User clicks checkbox next to todo
2. Mutation toggles `isCompleted` status
3. Convex updates database record
4. Real-time query reflects change
5. Visual feedback (strikethrough, color change)
6. Counter updates to show new active count

### Filtering Todos
1. User clicks filter button (All/Active/Completed)
2. Client-side filter state updates
3. Derived value recalculates filtered list
4. View updates to show filtered todos
5. Empty state message if no matching todos
6. No server request needed (client-side only)

### Clearing Completed
1. User clicks "Clear completed (N)" button
2. Mutation queries for all completed todos
3. Convex deletes all matching records
4. Real-time query updates list
5. Button disappears (no completed todos left)
6. Active count remains unchanged
